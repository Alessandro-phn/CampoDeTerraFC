# Contribuindo com Campo de Terra FC

## Convenções

- **Branches:** `feature/nome`, `fix/nome`, `refactor/nome`
- **Commits:** Em português, imperativo — `Adiciona sistema de passe`, `Corrige bug na física da bola`
- **PRs:** Descrição clara do que muda e por quê
- **Code Review:** Pelo menos 1 aprovação antes de merge

## Estilo de Código

- Siga as convenções em `TechnicalArchitecture.md`
- Todo código deve ter comentários XML doc
- Nenhuma função vazia ou TODO sem issue associada
- Testes para lógica de negócio
